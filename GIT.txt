1. Для початку потрібно ініціювати git репозиторій в каталозі де створений проект, оскільки це не було виконано на початку. Виконуємо це за допомогою команди git init та заповнюємо за необхідністю запропоновані поля. 
2. Після чого потрібно додати усі наші створені файли в так звану staging area (команда git add .), де зберігаються файли із змінами до того як буде створено коміт з усіма файлами або їх частиною. 
3. Після чого створюємо коміт git commit -m “тут вказуємо опис даного коміту, зазвичай це має бути щось коротке, що найкраще відображає сенс”. Відбувається перенесення наших файлів з staging area в локальний репозиторій. 
4. Далі додаємо віддалений репозиторій за допомогою команди git remote add origin git@example.com:example/test.git 
5. Після чого можна відправити наші файли з локального репозиторію до віддаленого, команда: git push -u origin master і файли з’являться в репозиторії клієнта.
